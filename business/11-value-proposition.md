## The SLIPPERY Framework for Persuasive Copy

The **SLIPPERY Framework** creates copy that keeps readers engaged from start to finish:

```
┌─────────────────────────────────────────────────────────────────┐
│                     THE SLIPPERY FRAMEWORK                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   S - SHORT SENTENCES                                           │
│       Keep sentences punchy. Easy to read. Easy to remember.    │
│       → "GPU costs are crushing AI innovation. We fix that."    │
│                                                                 │
│   L - LOGICAL FLOW                                              │
│       Guide readers step-by-step. No confusion.                 │
│       → Problem → Solution → How → Why us → CTA                 │
│                                                                 │
│   I - INTERESTING HOOKS                                         │
│       Open loops. Spark curiosity. Make them NEED to read on.   │
│       → "What if your gaming PC could pay for itself?"          │
│                                                                 │
│   P - PERSONAL CONNECTION                                       │
│       Show you understand them. Build relatability.             │
│       → "We were ML engineers who couldn't afford AWS."         │
│                                                                 │
│   P - PROOF & CREDIBILITY                                       │
│       Back claims with evidence. Remove doubt.                  │
│       → Benchmarks, testimonials, partnerships, numbers         │
│                                                                 │
│   E - EMOTIONAL TRIGGERS                                        │
│       Tap into feelings that drive action.                      │
│       → Frustration, excitement, fear of missing out            │
│                                                                 │
│   R - REPETITION OF KEY MESSAGE                                 │
│       Say your core message multiple times, different ways.     │
│       → "80% cheaper" in headline, body, proof, CTA             │
│                                                                 │
│   Y - YOUR READER FOCUS                                         │
│       Always "you", never "we". Their perspective, not yours.   │
│       → "You save money" not "We offer savings"                 │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### SLIPPERY Checklist

```
BEFORE PUBLISHING ANY COPY:
────────────────────────────

□ S - SHORT SENTENCES
  □ Average sentence under 15 words?
  □ No run-on paragraphs?
  □ Can be scanned quickly?

□ L - LOGICAL FLOW
  □ Clear progression from problem to solution?
  □ No confusing jumps or jargon?
  □ Reader always knows what comes next?

□ I - INTERESTING HOOKS
  □ First line creates curiosity?
  □ Open loops throughout?
  □ Reader wants to keep going?

□ P - PERSONAL CONNECTION
  □ Shows understanding of their situation?
  □ Relatable language and examples?
  □ Feels human, not corporate?

□ P - PROOF & CREDIBILITY
  □ Claims backed by evidence?
  □ Social proof included?
  □ Specific numbers, not vague claims?

□ E - EMOTIONAL TRIGGERS
  □ Taps into pain points?
  □ Creates excitement about solution?
  □ Motivates action?

□ R - REPETITION OF KEY MESSAGE
  □ Core benefit stated 3+ times?
  □ Different angles, same message?
  □ Memorable and sticky?

□ Y - YOUR READER FOCUS
  □ More "you" than "we"?
  □ Benefits, not features?
  □ Reader is the hero, not you?
```

## The 4U Framework for Headlines

Every headline and value proposition should be evaluated against the **4U Framework**:

```
┌─────────────────────────────────────────────────────────────────┐
│                        THE 4U FRAMEWORK                         │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   1. USEFUL                                                     │
│      Does it provide a clear benefit to the reader?             │
│      → "What's in it for me?"                                   │
│                                                                 │
│   2. URGENT                                                     │
│      Does it create a reason to act NOW?                        │
│      → "Why should I care today?"                               │
│                                                                 │
│   3. UNIQUE                                                     │
│      Is it differentiated from alternatives?                    │
│      → "Why you and not someone else?"                          │
│                                                                 │
│   4. ULTRA-SPECIFIC                                             │
│      Does it use concrete numbers and details?                  │
│      → "Prove it with specifics"                                │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 4U Checklist (Use Before Publishing)

```
BEFORE PUBLISHING ANY HEADLINE OR VALUE PROP:
─────────────────────────────────────────────

□ USEFUL      — Does it promise a clear, desirable benefit?
               Not features, but outcomes the reader wants.

□ URGENT      — Is there a compelling reason to act now?
               Time-sensitive, opportunity cost, or pain point.

□ UNIQUE      — Could a competitor say the exact same thing?
               If yes, add differentiation.

□ ULTRA-SPECIFIC — Are there real numbers, timeframes, or details?
                   Vague claims are forgettable. Specifics stick.

SCORING:
├── Rate each U from 1-5
├── Total: Aim for 16+ out of 20
├── Below 14: Rewrite before using
└── 18+: Ready for prime time
```
